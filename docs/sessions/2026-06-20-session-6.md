## Root Causes Identified

1. **Ruff linter (492 errors):** Multiple ruff rules were misconfigured:
   - `S105` (hardcoded passwords) fired in test files using test secrets
   - `S311` (pseudo-random crypto) fired on OTP generator
   - `E501` (line too long) fired on SQL strings and comment lines in tests
   - `PLR2004`, `PLW0603`, `ARG001/002`, `PLC0415`, `B008`, `B904`, `SIM102` etc. — all valid patterns in FastAPI microservice architecture
   - Ruff configuration used deprecated top-level keys (`select`, `ignore`) instead of `[tool.ruff.lint]`

2. **Unit test job collected only 20 tests (JWT):** The CI command was `pytest tests/unit -v --cov=packages` with a single `PYTHONPATH=$GITHUB_WORKSPACE`. However, the other unit test subdirectories (`academic/`, `workflow/`, `mdm/`, `assets/`, `logbook/`) import from `app.*` which is service-specific. Without the service path in PYTHONPATH, collection failed with `ModuleNotFoundError: No module named 'app'`.

3. **Coverage below 80%:** Because only 20 JWT tests ran and collected only `packages/shared` coverage, total coverage was 52% — below the 80% `fail_under` threshold.

4. **Python syntax error:** `services/snx-logbook/app/routers/logbook.py` had a parameter-without-default following a parameter-with-default (`phase: Optional[str] = None` before `current_user: Annotated[...]`), which caused `black --safe` to fail.
