## Decisions Made

- Placed all backend services under a secondary `services` compose profile in `docker-compose.yml` to prevent local resource footprint inflation while keeping local database engines running instantly.
- Handled Python module naming namespace overlap (`app/` package collision) by separating test files and running them with independent `PYTHONPATH` parameters, matching production service deployment isolation.

## Issues Encountered

- **`NoReferencedTableError`**: SQLAlchemy could not compile foreign key mappers for `academic_years.tenant_id` because `Tenant` model was not declared in `snx-academic`'s metadata. Resolved by defining a minimal `Tenant` reference model in `snx-academic` and `snx-institution`.
- **Unique Constraint & Idempotency**: Previous failed tests left seeded records in test database, violating constraints on repeat runs. Resolved by adding a tenant-scoped data cleanup block at the beginning of each integration test.
- **Missing Mapped Columns**: Defining properties dynamically in test files without mapping them in models caused SQLAlchemy to omit columns like `academic_year_id` from generated SQL inserts. Fixed by defining all required database columns as mapped properties in the local reference models.
