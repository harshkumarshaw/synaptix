1. **`audit_log` Table Check Constraint**: Encountered `CheckViolationError` during result submission writes to `audit_log`. Standardized all actions (`submit_result` -> `SUBMIT_RESULT`, `record_moderation` -> `RECORD_MODERATION`, `generate_mark_sheet` -> `GENERATE_MARK_SHEET`, etc.) to uppercase.

## Lessons Learned
- **audit_log Action Constraints**: `audit_log.action` column has a strict DB-level check constraint `action ~ '^[A-Z_]+$'` that will reject any lowercase strings. Always use uppercase action names (e.g. `CREATE`, `UPDATE`, `SUBMIT_RESULT`).
- **weasyprint & qrcode Dependencies**: Installing WeasyPrint requires adding `weasyprint` and `qrcode[pil]` to the environment.
