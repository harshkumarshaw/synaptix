## Decisions Made

- Styled the application by mapping HSL theme variables to CSS custom variables in `tailwind.config.ts`, maintaining strict compatibility with Tailwind CSS v3.
- Utilized the `render` prop in the `SidebarMenuButton` shadcn component rather than `asChild` since the generated shadcn code was utilizing Base UI's custom button primitives.
- Added a Developer Bypass button bar to `login-form.tsx` allowing one-click login for Admin, Faculty, and Student roles using local mock JWT tokens to enable offline/local validation without requiring FastAPI services to be active.

## Issues Encountered

- **Tailwind Compatibility**: The shadcn init command generated Tailwind v4-style `@import` and variables. Handled by resetting `globals.css` to standard `@tailwind` directives and configuring `tailwind.config.ts` accordingly.
- **Form Component missing from base-nova registry**: Resolved by switching `components.json` style to `default`, which allows pulling the standard shadcn Form primitives successfully.
- **Unescaped entity in faculty-dashboard**: Fixed by replacing `'` with `&apos;`.
- **SidebarMenuButton typed props**: Fixed by passing `render={<Link ... />}` instead of `asChild`.
