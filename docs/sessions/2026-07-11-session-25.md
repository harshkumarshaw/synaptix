## Work Done

### Root Cause Analysis
1. **`aggregate_ia()` returning 0 for logbook_marks**: The logbook query was filtering by `LogbookAssessment.subject_code == course.code`. Since `course.code` holds a unique suffixed value (e.g. `ANAT_ee68c1`), it never matched the logbook's `subject_code` (`ANAT`). Fix: use `course.subject_code` instead.

2. **FK violation on clinical_evaluations.evaluator_id**: `clinical_evaluations.evaluator_id` references `faculty.user_id` (the user's UUID), NOT `faculty.id`. Tests were passing `faculty_id` (the faculty row PK) when they needed to pass `fac_user_id` (the linked user's UUID). Fix: `seed_faculty_with_user()` now returns `user_id`, and all test INSERT statements use `fac_user_id`.

3. **FK violation on exam_eligibility (cross-tenant test)**: `check_student_eligibility()` was writing an eligibility row with `other_tenant_id` even when the student didn't exist in that tenant. Fix: added explicit tenant isolation checks at the top of the function.

4. **`generate_hall_ticket()` ignoring overrides**: After `override_eligibility()` set `is_eligible=True`, calling `generate_hall_ticket()` re-ran `check_student_eligibility()` which recomputed eligibility as False (no attendance etc.) and raised error. Fix: check stored eligibility record first; only recompute if no stored record or no override flag.

5. **`check_student_eligibility()` logbook/prerequisite**: Same `course.code` vs `course.subject_code` bug. Fixed both the logbook query and the `get_prerequisites()` call.

6. **test_nmc_compliance.py missing `select` import**: `select` from SQLAlchemy was not imported. Fixed.

7. **test_exm_nmc_004 AETCOM query by `code`**: The seed helper appends a UUID suffix to `code`, so querying by `course.code == "AETCOM"` never matches. Fixed to query by `course.subject_code == "AETCOM"`.

8. **seed helpers producing duplicate key errors**: `academic_years`, `batches`, `students`, `users` were using hardcoded values like `"student@jmn.edu.in"` and `"R123"` with `ON CONFLICT DO NOTHING`, silently colliding and returning NULL for IDs. Fixed by generating UUID-suffixed unique values for every seed run, removing silent conflict suppressions.

## Lessons Learned
- `course.code` in snx-academic always has a unique suffix appended. Only `course.subject_code` holds the canonical subject abbreviation (e.g. `ANAT`, `PATH`). All service-level queries against logbooks and prerequisites should use `subject_code`.
- FK `clinical_evaluations.evaluator_id → faculty(tenant_id, user_id)` — always pass the `user_id`, not the `faculty.id` row PK.
- FK `practical_assessments.examiner_id → faculty(tenant_id, user_id)` — same rule.
- FK `viva_scores.examiner_id → faculty(tenant_id, user_id)` — same rule.
- Test seed helpers must NEVER use hardcoded unique values with `ON CONFLICT DO NOTHING`. Always use `uuid.uuid4().hex[:N]` suffixes.
