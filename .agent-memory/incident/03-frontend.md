# 03-frontend — Incident Memory

Failures, near-misses, and what NOT to do.

## Incidents

- **Missing components in base-nova registry**: Initializing shadcn with the defaults `--preset=base-nova` wrote a config pointing to an registry subset lacking standard components (like `form` or `toast`).
  - *Fix:* Modify `components.json` style to `default` or re-initialize shadcn with the standard configuration.

## Things That Failed

- **Next.js Build Failure on globals.css**: Shadcn CLI generated a `globals.css` with `@import "shadcn/tailwind.css"` and `@import "tw-animate-css"`. Since Next.js is configured with Tailwind v3, these imports caused compiler/parser failures.
  - *Fix:* Replaced imports with standard `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;` directives and mapped HSL colors to custom properties in `tailwind.config.ts`.

## Things to Avoid

- Avoid using the `asChild` prop on `SidebarMenuButton` components generated by newer shadcn versions. Base UI utilizes custom render helpers which conflict with Radix UI's standard `asChild` prop.
